12. What's new v{{ version }}

A history of feature releases. See the full CHANGELOG.md on GitHub.

1.2.6
  • Fixed a build-time warning — the section reorder drag placeholder is now correctly imported, preventing a possible break on a future Angular upgrade.
  • Fixed duplicate section names — two sections sharing the same name (e.g. two unnumbered "VERSE" sections) no longer risk mixing up editing state between them.
  • Faster chord repositioning — chord position calculations are now cached per line instead of recomputed from scratch on every render, speeding up transposition on chord-dense charts.

1.2.5
  • Tinted backgrounds per color theme — switching to Pink, Red, Amber, or Green now also tints the app background, surface, border, and hover colors with a subtle hue matching the accent. Works in both Light and Dark mode. Blue (default) keeps the neutral grey palette.

1.2.4
  • Manual page translations — all 11 content sections of this manual are now translated into all five app languages (Latin, Traditional Chinese, Bahasa Indonesia, Basa Jawa), each with a distinct humorous tone. The §12 changelog remains in English for easy editing.

1.2.3
  • Color themes — five accent color themes now available in ⚙️ Settings → Appearance: Blue (default), Pink, Red, Amber, and Green. Each color has automatic light and dark mode variants. The color swatch picker and brightness toggle are independent controls.
  • Multi-language UI — the app now supports five languages: English, 🏛️ Latina, 🀄 繁體中文, 🌴 Bahasa Indonesia, and 🌾 Basa Jawa. Switch using the pill buttons in ⚙️ Settings → Language or from the top of this manual page. Every language switch shows a toast notification. Language preference is saved and restored per set.
  • Funny app names — the home page title and tagline adapt per language: Arca Musica (Latin), 讚美神器 (Chinese), Koper Pujian (Indonesian), Piranti Puji-Pujian (Javanese).

1.2.2
  • Accidentals preference — ⚙️ Settings now has a ♭ Flats / Auto / ♯ Sharps control. Your choice applies everywhere at once: chord buttons in the editor, toolbar and sidebar key display, "Jump to" dropdown, and PDF/Markdown exports. Auto (default) uses conventional key-context spelling.

1.2.1
  • PDF font size — a new control in ⚙️ Settings sets the export font size independently (10–20 px, default 14 px). Sizes above 14 px switch automatically to single-column layout so text never overflows the page.
  • Fix: "Jump to" dropdown going blank — transposing to Db, Eb, Ab, or Bb could produce a mismatched enharmonic spelling (e.g. C# vs Db) that didn't match any dropdown option, leaving it blank. Key resolution now always returns the conventional flat spelling that matches the list.

1.2.0
  • 📤 Export modal — a header button opens a modal with three clearly-labelled export options (Song PDF, Set PDF, Markdown). The export buttons are removed from the editor toolbar.
  • ⚙️ Settings modal — a header button collects theme, text size, and Latin mode in one place. The font size and dark mode controls are removed from the header bar.
  • Larger text sizes — font size now goes up to 32 px (new steps: 24, 28, 32 px), useful for big-screen presentations.
  • Unified preferences — theme, font size, and Latin mode are now stored as one JSON object; existing preferences migrate automatically.
  • Fix: "Jump to" key dropdown — the dropdown now correctly shows the song's current key. Previously it always appeared to default to C, and selecting C when already in C had no effect.

1.1.14
  • Export buttons regrouped — the three export buttons are now grouped under a small "Export" label in the editor toolbar, making their purpose immediately clear. ↓ All PDF is renamed to ↓ Set PDF to match the "set" terminology used everywhere else in the app.
  • Beta link on landing page — a subtle link at the bottom of the upload page points to the beta deployment where new features land first.

1.1.13
  • Latin mode toast now auto-dismisses — the "Modus Latinus Activatus" toast previously required a tap elsewhere before disappearing. Switching to an Angular signal fixes the 3-second timer in zoneless mode.
  • Pencil affordance on song title — a ✎ icon appears beside the song title on hover (desktop) or faintly at all times (mobile/tablet), making it clear the title is editable.
  • Extended Latin coverage — Latin mode now translates the remaining UI strings: + Line, + chord, + note in the editor; Already in set: in the duplicate warning; and Text size in the header.

1.1.12
  • Latin Easter Egg — a 🏛️ Switch to Latin pill button (right here in this manual) flips every UI label in the app to humorous Latin. A toast notification confirms activation. Latin mode is saved with each set and restored when that set is loaded.
  • Cantus Secretus — activate Latin mode to unlock a hidden collapsible section on this page containing the Latin lyrics of Unus Angelus Alae (Nobuo Uematsu, 1997). 🏛️

1.1.11
  • Easier to find where sets live — the header now shows the name of the set you're working in (e.g. 💾 Sunday Service ▾, or 💾 Untitled set ▾ when nothing is saved) and a dedicated ➕ New button starts a fresh set in one click. On mobile these collapse to compact icons to save space.
  • Consistent wording — what used to be called a "session" is now called a set throughout the app, matching the "Songs in Set" list.

1.1.10
  • Songs without a Tempo field now split correctly — PDFs where a song's metadata line omits the tempo (e.g. Key - D | Time - 4/4) were not recognised as a new song boundary, causing subsequent pages to be merged into the previous song. The meta-line pattern now accepts tempo as optional.

1.1.9
  • PDF upload and append-PDF now render immediately — the same change-detection gap fixed in 1.1.8 for Sessions-panel import also affected uploading a PDF from the landing page, loading a .wt file via "📂 Load session file," and appending a PDF via "+ Import PDF" inside the editor. All three now render the editor instantly instead of requiring an extra tap or click elsewhere.

1.1.8
  • Session file import now loads immediately — importing a .wt file from the Sessions panel now navigates to the editor instantly; previously the view would not update until you tapped or clicked elsewhere.

1.1.7
  • PDF import fixed on iOS and Android — uploading a PDF no longer freezes on the "Parsing PDF…" spinner. Two root causes were fixed: PDF.js was downgraded from v5 to v4 (v5's streaming text-extraction API hangs indefinitely inside iOS Safari web workers), and the PDF.js worker script is now fetched and inlined in a self-contained Blob URL, removing a hidden cross-origin request that was silently failing on GitHub Pages and broke Android.
  • TURNAROUND recognized as a section — TURNAROUND is now parsed as a section header (alongside VERSE, CHORUS, BRIDGE, etc.) rather than treated as lyric text.

1.1.6
  • Add-songs UI redesigned — on desktop and tablet, + New Song and + Import PDF now appear directly as the last item in the song list — no toggle needed.
